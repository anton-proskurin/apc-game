# APC - Архитектура проекта

## Обзор

APC — браузерная игра-симулятор взрослой жизни для ребёнка (8 лет). Vanilla HTML + CSS + JS (ES6 modules), без фреймворков и сборщиков.

## Стек технологий

- **Frontend:** HTML5, CSS3, JavaScript (ES6 modules)
- **Хранение:** localStorage
- **Сервер разработки:** `python3 -m http.server 8080`
- **Деплой:** GitHub Pages (статический сайт)

## Архитектурные решения

### 1. Паттерн сцен

Каждая сцена — ES6-модуль, экспортирующий:
- `render(container, params)` — рисует содержимое сцены
- `cleanup()` — очищает таймеры, обработчики событий

Router управляет жизненным циклом: при навигации вызывает `cleanup()` текущей сцены, очищает контейнер, вызывает `render()` новой. Автоматически сохраняет состояние при каждом переходе.

### 2. Центральное состояние (state.js)

Единый объект `gameState` — source of truth. Все мутации через хелперы (`addMoney`, `changeStat`, `addPet` и т.д.), которые генерируют события через встроенный event emitter.

### 3. Event-driven UI

Компоненты подписываются на события (`statsChange`, `inventoryChange` и т.д.) для обновления без полного ре-рендера. Пример: HUD обновляется при каждом изменении статов.

### 4. Игровое время (time.js)

- 1 тик = 3 секунды реальных = 1 час игровой
- Деградация: голод -1/тик, счастье падает при голоде < 20
- Питомцы: голод -2, счастье -1, чистота -1 за тик
- Ночью (22:00-06:00) энергия восстанавливается +2
- Дети взрослеют каждые 30 игровых дней

### 5. Навигация

Hash-based SPA роутер. URL вида `#scene-id`. Кнопка "Назад" в браузере работает. Нижняя навигация фиксирована (карта, работа, дом, питомцы, друзья).

### 6. Мобильный дизайн

- Max-width: 480px, центрирование
- Все кнопки >= 44x44px (Apple HIG)
- Touch-action: manipulation (отключает zoom по двойному тапу)
- Safe area insets для iPhone
- `-webkit-overflow-scrolling: touch`

## Потоки данных

```
Пользователь тапает → scene.js вызывает state helper →
state helper мутирует gameState → emit событие →
ui.js обновляет HUD → storage.js автосохраняет через 30 сек
```
